GIT LÀ GÌ? CÁC LỆNH CƠ BẢN CỦA GIT

Các lệnh git cơ bản
1) git config
Tác dụng: Để set username và email của bạn trong main configuration file. Cách xài: Để kiểm tra tênvà kiểu email trog cấu hình dùng git config --global user.name và git config --global user.email. Để set email hoặc tên mới git config--global user.name = "Hải Nguyễn" và git config --global user.email = "hainguyen@gmail.com"

2) git init Tác dụng: Khởi tạo 1 git reponsitory 1 project mới hoặc đã có
Cách xài: git init trong thư mục gốc của dự án

3) git clone Tác dụng: Copy 1 git repository từ remote source.
Cách xài: git clone <:git clone url:>

4) git status Tác dụng: Để check trạng thái những file bạn đã thay đổi trong thư mục làm việc. VD: Tất cả những thay đổi từ lần commit cuối cùng.
Cách xài: git status trong thư mục làm việc

5) git add Tác dụng: Thay đổi đến stage/index trong thư mục làm việc
Cách xài: git add

6) git commit Tác dụng: commit nghĩa là một action để Git lưu lại một snapshot của các sự thay đổi trong thư mục làm việc. Và các tệp tin, thư mục thay đổi phải nằm trong Staging Area. Mỗi lần commit nó sẽ được lưu lại lịch sử của code kèm theo tên và địa chỉ email của người commit. Ngoài ra trong git bạn cũng có thể khôi phục lại tệp tin trong lịch sử commit của nó để chia cho 1 branch khác, vì vậy bạn sẽ dễ dàng khôi phục lại tệp tin trước đó.
Cách dùng: git commit -m "Đây là message, bạn dùng để note những thay đổi để sau này dễ dò lại"
7) git push/git pull Tác dụng: push hoặc pull các thay đổi đến remote. Nếu bạn đã added và committed các thay đổi và bạn muốn đẩy nó lên hoặc remote của bạn đã update và bạn apply tất cả những thay đổi đó trên code của mình

8) git remote Tác dụng: Để check remote/source bạn có hoặc add thêm remote
Cách xài: git remote dùng để kiểm tra và liệt kê. Và git remote add <:remote_url:> để thêm.

9) git add Tác dụng: để đưa một tập tin vào Staging Area.
Cách dùng: git add tên_file hoặc muốn thêm hết các file của thư mục thì git add all.
